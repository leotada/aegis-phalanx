   - Run the entire test suite. Correct issues until everything passes.
   - Refactor ensuring Clean Code.

4. PEER REVIEW & PR:
   - Make semantic commits (Conventional Commits).
   - Push the branch to origin (`git push origin <branch>`).
   - Use the GitHub CLI (`gh pr create`) to open a Pull Request detailing the implementation and test coverage.
   - Review your own PR via `gh pr view` or by reading the diff. If you find gaps not covered by tests, fix them, make a new commit, and update the PR.
